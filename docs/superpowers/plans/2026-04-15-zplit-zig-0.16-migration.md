# zplit Zig 0.16.0 Migration — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** zplit を Zig 0.15.x から Zig 0.16.0 に移行し、v0.2.0 として release-ready 状態にする。挙動は1ビットも変えない pure migration。

**Architecture:** `main` を 0.16 の "Juicy Main" 形式 (`pub fn main(init: std.process.Init)`) に変更し、`Server` / `Client` 構造体に `io: std.Io` フィールドを持たせて内部メソッドから `self.io` で参照する。`std.fs.*Absolute` は `std.Io.Dir` 経由に置換、`std.posix.*` の raw syscall は維持。

**Tech Stack:** Zig 0.16.0 標準ライブラリ。外部依存なし。

**Spec:** `docs/superpowers/specs/2026-04-15-zplit-zig-0.16-design.md`

**前提:** Zig 0.16.0 がローカルマシンにインストールされていること（`zig version` で `0.16.0` が出ること）。降臨前は P0 のみ実行可能。

## Phase 2: main.zig を Init 形式に切り替え

**Files:**
- Modify: `src/main.zig`

`pub fn main(init: std.process.Init) !void` 形式に変更し、`gpa` / `io` / `args` / `environ` を `init` 経由に統一。

- [ ] **Step 2.1: main シグネチャ変更**

  `src/main.zig` の `pub fn main()` を：
  ```zig
  pub fn main(init: std.process.Init) !void {
      const gpa = init.gpa;
      const io = init.io;
      // ... 既存処理。allocator は gpa を使用。
  }
  ```

  既存の `var gpa = std.heap.GeneralPurposeAllocator(.{}){}; defer _ = gpa.deinit(); const allocator = gpa.allocator();` は削除。

- [ ] **Step 2.2: process.argsAlloc を Init.minimal.args に置換**

  src/main.zig:407 周辺：
  ```zig
  // OLD: const args = try std.process.argsAlloc(allocator);
  // NEW:
  const args = init.minimal.args;
  ```

  対応する `defer std.process.argsFree(allocator, args);` も削除。

- [ ] **Step 2.3: std.os.environ を Init 経由に置換**

  src/main.zig:267 周辺：
  ```zig
  // OLD: const env_slice = std.os.environ;
  // NEW:
  const env_slice = init.minimal.environ;
  ```

  型差分（`[][*:0]u8` vs `[*:null]const ?[*:0]const u8`）に注意。コメントを更新。

- [ ] **Step 2.4: std.fs.selfExePathAlloc を std.process.executablePathAlloc に置換**

  src/main.zig:256：
  ```zig
  // OLD: const self_exe = try std.fs.selfExePathAlloc(allocator);
  // NEW:
  const self_exe = try std.process.executablePathAlloc(gpa);
  ```

- [ ] **Step 2.5: std.Thread.sleep を io.sleep に置換**

  src/main.zig:304：
  ```zig
  // OLD: std.Thread.sleep(10 * std.time.ns_per_ms);
  // NEW: try io.sleep(10 * std.time.ns_per_ms);
  ```
  実機の API 名で要調整（`io.sleep` / `std.Io.Timestamp.sleep` 等）。

- [ ] **Step 2.6: io と gpa を後続関数に伝播**

  `runServer` / `runClient` / `attachExisting` / その他 main から呼ぶ関数の引数に `io: std.Io` を追加。Server/Client が IO 必要なら `Server.init` / `Client.init` にも追加。

- [ ] **Step 2.7: P2 ビルド確認**

  Run: `zig build 2>&1 | head -40`
  Expected: main.zig 起因のエラーが消える（fs/posix の残エラーは P3 以降）。

- [ ] **Step 2.8: P2 commit**

  ```bash
  git add src/main.zig
  git commit -m "refactor(main): adopt Zig 0.16 Init-style main entrypoint"
  ```

## Phase 5: std.fs.*Absolute → std.Io.Dir 移行

**Files:**
- Modify: `src/main.zig`, `src/session.zig`, `src/config.zig`, `src/server.zig`

最大のPhase。`io` 引数を伝播しながら fs API を置換。

- [ ] **Step 5.1: session.zig の load/save に io 引数追加**

  `session.save(allocator, ...)` → `session.save(io, allocator, ...)` のように全公開関数に `io: std.Io` を追加。

- [ ] **Step 5.2: session.zig の fs 操作を std.Io.Dir に置換**

  対象:
  - line 109: `std.fs.makeDirAbsolute(data_dir)` → `std.Io.Dir.createDirPath(io, data_dir)`
  - line 120: `std.fs.createFileAbsolute(session_path, .{ .truncate = true })` → `std.Io.Dir.cwd().createFile(io, session_path, .{ .truncate = true })`（パス絶対なので問題なし）
  - line 122, 286: `file.writeAll(json)` → `file.writeStreamingAll(io, json)`
  - line 134, 291: `std.fs.openFileAbsolute(...)` → 同上 createFile 同様 openFile 経由
  - line 273, 308, 348: `std.fs.deleteFileAbsolute(...)` → `std.Io.Dir.cwd().deleteFile(io, ...)`
  - line 284: `std.fs.createFileAbsolute(path, .{})` → 同上
  - line 305, 336: `std.time.milliTimestamp()` → `std.Io.Timestamp.now().toMillis()` 相当（実機で API 確認）
  - line 341: `std.fs.createFileAbsolute(file_path, .{})` → 同上
  - line 345, 351: `std.fs.accessAbsolute(file_path, .{})` → `std.Io.Dir.cwd().access(io, file_path, .{})`

  実際の0.16 API名は `zig std` で確認しながら進める。

- [ ] **Step 5.3: config.zig の fs 操作を移行**

  - Config.load に `io: std.Io` 引数追加
  - line 144: `std.fs.openFileAbsolute(path, .{})` → `std.Io.Dir.cwd().openFile(io, path, .{})`
  - line 140: `std.posix.getenv("HOME")` は維持（posix raw 維持方針）

- [ ] **Step 5.4: server.zig の fs 操作を移行**

  - line 207: `std.fs.path.dirname(socket_path)` → `std.Io.Dir.path.dirname(socket_path)`（0.15互換層が残れば変更不要、要実機確認）
  - line 208: `std.fs.makeDirAbsolute(dir_path)` → `std.Io.Dir.createDirPath(io, dir_path)`
  - Server に `io: std.Io` フィールド追加し、`Server.init` 引数に追加

- [ ] **Step 5.5: main.zig の fs 操作を移行**

  - line 76, 197, 246, 305, 337: `std.fs.accessAbsolute(socket_path, .{})` → `std.Io.Dir.cwd().access(io, socket_path, .{})`
  - line 103, 150: `std.fs.openDirAbsolute(dir_path, .{ .iterate = true })` → `std.Io.Dir.cwd().openDir(io, dir_path, .{ .iterate = true })`

- [ ] **Step 5.6: 呼び出し側の調整**

  Session/Config/Server.init を呼ぶ箇所すべてに `io` を渡す。

- [ ] **Step 5.7: ビルド確認**

  Run: `zig build 2>&1 | head -40`
  Expected: fs 系エラー大幅減。残るは Writer 系と stdout 系。
